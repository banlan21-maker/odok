# 오독오독 개선 과제

> 마지막 업데이트: 2026-04-05 | 현재 버전: v2.0.1

---

## 🔴 1순위 — 리텐션/핵심 UX

### ✅ [완료] 하이라이트·공유 기능 Android 미작동
- **원인**: Android ActionMode(네이티브 텍스트 선택 툴바)가 WebView 위에서 커스텀 메뉴를 가림
- **해결**: `MainActivity.java`에서 `onWindowStartingActionMode` override로 차단

### 🚧 [진행중] 푸시 알림 시스템 없음
- 내 책에 댓글, 좋아요, 팔로우가 와도 앱을 직접 열어봐야만 알 수 있음
- 리텐션에 직결되는 가장 큰 구멍
- **필요 작업**:
  - `@capacitor/push-notifications` 설치
  - FCM 토큰 Firestore 저장 (users/{uid}/fcmToken)
  - Cloud Functions 트리거: 댓글 작성 → 책 작가에게 알림
  - Cloud Functions 트리거: 좋아요 → 책 작가에게 알림
  - Cloud Functions 트리거: 팔로우 → 팔로우 대상에게 알림

### ⬜ 읽기 진행률 저장 없음
- 책을 읽다 나갔다 돌아오면 처음부터 다시 읽어야 함
- `completions`(완독) 기록은 있으나 중간 위치 저장 없음
- **필요 작업**: 스크롤 위치 or 페이지 인덱스를 Firestore에 저장

---

## 🟠 2순위 — 데이터 신뢰성

### ⬜ 댓글수(commentCount) 필드 신뢰성 문제
- 책 목록/미리보기에서 `book.commentCount` 필드로 표시하나,
  댓글은 별도 서브컬렉션(`book_comments`)에서 관리
- 댓글 추가/삭제 시 book 문서의 `commentCount`를 동기화하는 로직 확인 필요
- **필요 작업**: Cloud Function 또는 클라이언트 코드에서 카운터 동기화 확인/추가

### ⬜ 구버전 APK 강제 업데이트 불가
- v1.0.9 이하 사용자는 버전체크 코드 자체가 없어 강제 업데이트 화면을 볼 수 없음
- v2.0.0 이상 사용자에 대한 다음 버전 강제 업데이트는 정상 작동
- **대응**: Firestore `app_settings/version_info.min_version` 관리, Play Store 자동 알림 의존

---

## 🟡 3순위 — 성능

### ⬜ 전체 책 데이터를 한 번에 로드
- `books` 배열 전체를 Firestore에서 로드 후 클라이언트에서 filter/sort
- 책이 수백 권 이상이면 로드 시간·메모리 문제 발생
- **필요 작업**: Firestore 쿼리 기반 페이지네이션 (limit + startAfter)

### ⬜ 검색 품질 한계
- Firestore는 부분 문자열 full-text 검색 미지원
- 현재 클라이언트 `Array.filter + includes()`로 검색 → 전체 로드 필요
- **필요 작업**: Algolia / Typesense 연동 또는 Firestore 배열 인덱스 패턴 적용

### ⬜ 공유 이미지 생성 속도
- `html2canvas` 는 모바일에서 느림 (특히 배경 이미지 + 텍스트 조합)
- **필요 작업**: Canvas API 직접 사용하거나 서버사이드 이미지 생성 검토

---

## 🟢 4순위 — 기능 추가

### ⬜ 독서 히스토리/통계
- 내가 읽은 책 목록, 이번 달 몇 권 읽었는지 등 통계 화면 없음
- `completions` 데이터는 이미 있어서 구현 용이

### ⬜ 알림 내역 화면
- 푸시 알림 구현 후 앱 내 알림함(인박스) 화면 필요

### ⬜ 책 쓰기 중 임시저장 명확화
- 장문 작성 중 실수로 뒤로가면 내용이 날아갈 수 있음
- 자동저장 상태 표시 또는 이탈 시 경고 다이얼로그 필요
